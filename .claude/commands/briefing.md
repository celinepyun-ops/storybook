오늘의 브리핑을 보여줘. 내가 놓친 것, 나한테 온 것, 지금 해야 할 것을 정리.

## 절차

1. 현재 사용자 파악:
   - git config user.name으로 이름 확인
   - "robin" 또는 "celine"으로 매핑

2. **읽지 않은 공지** 확인:
   - `.traces/announcements/` 폴더의 모든 파일을 읽어
   - 현재 사용자가 작성한 게 아닌 것만 필터
   - 날짜 내림차순으로 정리
   - `action-needed` 우선, `info`는 그 다음

3. **팀원들의 최근 작업** 확인:
   - `.traces/` 에서 다른 팀원 폴더의 최근 trace 파일 (최근 3일)
   - "팀원한테 공유할 것" 섹션만 추출

4. **현재 진행 상황** 파악:
   - git log --oneline -10으로 최근 커밋 확인
   - git branch로 현재 브랜치 확인
   - 열린 PR 확인: gh pr list --state open

5. **브리핑 출력** (아래 형식으로):

```
📋 오늘의 브리핑 — [날짜]
현재 브랜치: [브랜치명]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔔 공지사항
[공지가 있으면]
  • [action-needed] robin (3/14): 온보딩 플로우 2단계로 줄임. 피그마 업데이트 필요
  • [info] celine (3/13): 카드 컴포넌트 border-radius 변경

[없으면]
  새로운 공지 없음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

👥 팀원 활동 (최근 3일)
[다른 팀원의 trace "팀원한테 공유할 것" 내용 요약]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📌 열린 PR
  • #42 design/dashboard-redesign → staging (Celine)
  • #41 fix/quota-bug → staging (Robin)

[없으면]
  열린 PR 없음

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 이어서 할 것
[현재 사용자의 가장 최근 trace에서 "다음에 이어서 할 것" 추출]
```

## 주의
- 출력만 하고 파일은 생성하지 마
- 공지 읽었다고 표시하거나 삭제하지 마 (공지 정리는 작성자가 직접)
- 간결하게. 브리핑은 30초 안에 읽을 수 있어야 함
